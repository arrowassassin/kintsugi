INSTALL

Get the binaries, then wire your agent. One command does both.

> 1 · install + wire

One line, no clone — downloads prebuilt binaries (checksum-verified) or builds from source if your platform has no prebuilt: 

curl -fsSL https://github.com/arrowassassin/kintsugi/releases/latest/download/install.sh | sh

Or straight from source with Cargo (also no clone):

cargo install kintsugi

Note: cargo install gives you the binaries only — it doesn't fetch the optional local model (Kintsugi runs fine without one). Once it finishes, add the model anytime with kintsugi model install (builds the engine + downloads a GGUF; see below).

The installer then walks you through a short setup: wiring your agents (kintsugi init) and an optional local model. You can skip anything — Kintsugi works immediately with no model. To wire later or re-run it yourself: 

kintsugi init     # wire agents + start daemon      kintsugi stop   # stop it

Then run a command through any wired agent — or a raw shell with the shim dir on your PATH — and watch kintsugi log / kintsugi tui.

Works with any agent. kintsugi init detects Claude Code (~/.claude), Cursor CLI (~/.cursor), Codex CLI (~/.codex), Qwen Code (~/.qwen), Gemini CLI (~/.gemini), and Google Antigravity (~/.gemini/antigravity-cli): it wires Claude's native hook, prints the kintsugi-exec MCP server line for the rest, and the $PATH shim covers every other tool or raw shell-out (Aider, Continue, a Makefile, you). See docs/mcp.md.

  • Sub-second warnings; SAFE commands add negligible overhead.
  • Fail-open by default (a down daemon never blocks you); KINTSUGI_FAIL_CLOSED=1 to flip.
  • Panic kill-switch: kintsugi panic halts everything, kintsugi resume clears it.

> 1b · optional — pick a local model

Kintsugi works with no model (the offline heuristic scorer is always on). To also get a plain-English summary + risk score, the picker fetches a short, RAM-appropriate list of small instruct GGUF models, downloads your choice, and prints the one env var to set: 

curl -fsSL https://github.com/arrowassassin/kintsugi/releases/latest/download/pick-model.sh | sh

Or fold it into the install above with | sh -s -- --with-model. The query is constrained (filter=gguf, text-generation, sized to your RAM) so only viable options appear.

Manage it later with kintsugi model. Installed via cargo install and want the model? kintsugi model install builds the engine and downloads a model in one step. Already have a GGUF, or want a newer one? kintsugi model use /path/to/model.gguf points the daemon at it and reloads — any local model, no Kintsugi update, no recompile. kintsugi model status shows what's loaded (and why it might still be heuristic); kintsugi model remove reverts to the always-on heuristic scorer. The choice is persisted, so it survives daemon restarts without a shell env var.

> 2 · claude code plugin

Kintsugi ships as a Claude Code plugin + marketplace. The plugin is a thin wiring layer; install the binaries with the one-liner above. 

/plugin marketplace add arrowassassin/kintsugi
/plugin install kintsugi@kintsugi

It registers a PreToolUse hook (→ kintsugi-hook) and the kintsugi-exec MCP server (→ kintsugi-mcp). Ships disabled until you trust it.

> 3 · per-repo policy

Commit an .kintsugi.toml to tune Kintsugi per project: set the mode, pre-allow known-safe patterns, force-deny others, and set the unattended risk threshold. 

# .kintsugi.toml
mode = "attended"      # attended · unattended · notify

[rules]
allow = ["cargo run", "rm -rf target", "rm -rf node_modules"]
deny  = ["git push * --force", "kubectl * --context=prod"]

In unattended mode, ambiguous commands are denied and queued — only an allow rule (your decision) lets one auto-proceed. An allow never downgrades a rule-detected catastrophic command — that hard floor always stands.

> 4 · command cheat-sheet

kintsugi --help 
kintsugi init           detect agents, wire interception, start daemon + backstop
kintsugi stop           stop the background daemon + backstop (inverse of init)
kintsugi status         daemon / backstop / shim-on-PATH / log / chain / scorer
kintsugi guard <cmd>    launch an agent with interception forced on (yolo-safe)
kintsugi dry-run       what Kintsugi would've caught in your history (runs nothing)
kintsugi limits        the honest threat scope — what it can & can't protect
kintsugi update         check GitHub for a newer release & install in place
kintsugi model <cmd>    status · use <path> · pick · install · remove (local model)
kintsugi log [-n N]     recent command timeline
kintsugi tui            live, interactive timeline (filter: agent: session: since:)
kintsugi queue          commands held for approval
kintsugi approve <id>   release a held command (agent proceeds)
kintsugi deny <id>      reject a held command
kintsugi undo [--session] restore snapshotted files
kintsugi watch <path>   extend the FS-watcher backstop to more paths
kintsugi panic / resume  kill-switch on / off
kintsugi admin enforce-shell  lock the shim wiring in root-owned /etc files
                              (a normal user can't remove; root + admin pw can)
