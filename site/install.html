INSTALL

Get the binaries, then wire your agent. One command does both.

> 1 · install + wire

One line, no clone — downloads prebuilt binaries (checksum-verified) or builds from source if your platform has no prebuilt:

curl -fsSL https://arrowassassin.github.io/aegis/install.sh | sh

Or straight from source with Cargo (also no clone):

cargo install --git https://github.com/arrowassassin/aegis aegis-cli aegis-daemon aegis-intercept

Then wire your agents and start the daemon:

aegis init

Then run a command through any wired agent — or a raw shell with the shim dir on your PATH — and watch aegis log / aegis tui.

  • Sub-second warnings; SAFE commands add negligible overhead.
  • Fail-open by default (a down daemon never blocks you); AEGIS_FAIL_CLOSED=1 to flip.
  • Panic kill-switch: aegis panic halts everything, aegis resume clears it.

> 2 · claude code plugin

Aegis ships as a Claude Code plugin + marketplace. The plugin is a thin wiring layer; install the binaries with the one-liner above.

/plugin marketplace add arrowassassin/aegis
/plugin install aegis@aegis

It registers a PreToolUse hook (→ aegis-hook) and the aegis-exec MCP server (→ aegis-mcp). Ships disabled until you trust it.

> 3 · per-repo policy

Commit an .aegis.toml to tune Aegis per project: set the mode, pre-allow known-safe patterns, force-deny others, and set the unattended risk threshold.

# .aegis.toml
mode = "attended"      # attended · unattended · notify
threshold = 50         # unattended: deny ambiguous at/above this risk

[rules]
allow = ["cargo run", "rm -rf target", "rm -rf node_modules"]
deny  = ["git push * --force", "kubectl * --context=prod"]

An allow rule tames the ambiguous band but never downgrades a rule-detected catastrophic command — that hard floor always stands.

> 4 · command cheat-sheet

aegis --help
aegis init           detect agents, wire interception, start daemon
aegis status         daemon / socket / log / chain / kill-switch
aegis log [-n N]     recent command timeline
aegis tui            live, interactive timeline
aegis queue          commands held for approval
aegis approve <id>   release a held command (agent proceeds)
aegis deny <id>      reject a held command
aegis undo [--session] restore snapshotted files
aegis watch <path>   FS-watcher backstop for un-intercepted changes
aegis panic / resume  kill-switch on / off
